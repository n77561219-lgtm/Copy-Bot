# Команда агентов: Telegram Copywriter Bot

---

## Два контекста агентов

| Контекст | Когда работают | Кто входит |
|---|---|---|
| **Dev-агенты** | Разово, при разработке бота | Оркестратор, Архитектор, Разработчик, Декомпозитор, Security, Дебагер, Тестировщик |
| **Runtime-агенты** | При каждом запросе пользователя | Диспетчер, Аналитик стиля, Ресерчер, Копирайтер, Критик, Редактор, Контент-планер |

---

## Часть I — Runtime-агенты (работают внутри бота)

### Архитектура контентного pipeline

```
Пользователь
     │
     ▼
┌──────────┐
│ДИСПЕТЧЕР │  ← Haiku, каждое сообщение, роутинг
│  (Haiku) │
└────┬─────┘
     │
     ├─── WRITE_POST ──────────────────────────────┐
     │                                             │
     │    ┌─────────────┐    ┌───────────┐         │
     │    │  РЕСЕРЧЕР   │───▶│КОПИРАЙТЕР │         │
     │    │  (Sonnet)   │    │  (Sonnet) │         │
     │    └─────────────┘    └─────┬─────┘         │
     │                             │               │
     │                        ┌────▼────┐          │
     │                        │ КРИТИК  │◀─ loop   │
     │                        │ (Haiku) │  max 2x  │
     │                        └────┬────┘          │
     │                             │ REVISE        │
     │                        ┌────▼────┐          │
     │                        │РЕДАКТОР │          │
     │                        │(Sonnet) │          │
     │                        └─────────┘          │
     │                                             │
     ├─── EDIT_POST ──────────────────────────────▶┤
     │    Редактор → Критик → Пользователь         │
     │                                             │
     ├─── CONTENT_PLAN ───────────────────────────▶┤
     │    ┌────────────────┐                       │
     │    │КОНТЕНТ-ПЛАНЕР  │                       │
     │    │   (Sonnet)     │                       │
     │    └────────────────┘                       │
     │                                             │
     └─────────────────────────────────────────────┘
                                    │
                               Пользователь
```

### R-1. Диспетчер (Dispatcher)

| Параметр    | Значение                              |
|-------------|---------------------------------------|
| Модель      | `claude-haiku-4-5-20251001`           |
| Температура | `0.1`                                 |
| Роль        | Классификация интента, роутинг        |

**Описание:**
Первая точка входа для каждого сообщения. Классифицирует намерение
пользователя и извлекает параметры. Haiku — потому что задача
тривиальная и выполняется на каждый запрос: экономия критична.

**Интенты:** `WRITE_POST` / `EDIT_POST` / `CONTENT_PLAN` / `SHOW_PLAN` / `HELP` / `UNKNOWN`

**Выход:** `{ intent, topic, text, days }`

---

### R-2. Аналитик стиля (Style Analyst)

| Параметр    | Значение                              |
|-------------|---------------------------------------|
| Модель      | `claude-sonnet-4-6`                   |
| Температура | `0.1`                                 |
| Роль        | Извлечение авторского стиля           |

**Описание:**
Запускается один раз при загрузке примеров. Анализирует посты
из MD/JSON экспорта и строит `style_profile.json` — «ДНК стиля» автора.
Без него Копирайтер пишет обезличенный текст.

**Зоны ответственности:**
- Лексика: любимые слова, обороты, запрещённые слова
- Структура: длина предложений, абзацы, переносы строк
- Тональность: ирония, прямота, дерзость — с примерами из постов
- Форматирование: эмодзи, caps, тире, многоточия
- Антипаттерны: чего автор НИКОГДА не пишет
- Паттерны открытий и концовок постов

**Выход:** `style_profile.json`

---

### R-3. Ресерчер (Researcher)

| Параметр    | Значение                              |
|-------------|---------------------------------------|
| Модель      | `claude-sonnet-4-6`                   |
| Температура | `0.3`                                 |
| Роль        | Подготовка фактуры по теме            |

**Описание:**
Запускается перед Копирайтером. Собирает конкретику по теме:
факты, углы подачи, нестандартные точки зрения, идеи для крючка.
Без него Копирайтер пишет общими словами.

**Выход:** структурированный текст с: сутью темы, фактами/цифрами,
углами подачи, заблуждениями по теме, идеей для первого предложения.

---

### R-4. Копирайтер (Copywriter)

| Параметр    | Значение                              |
|-------------|---------------------------------------|
| Модель      | `claude-sonnet-4-6`                   |
| Температура | `0.75`                                |
| Роль        | Генерация постов в стиле пользователя |

**Описание:**
Пишет пост на основе `style_profile.json` + фактуры от Ресерчера.
Температура 0.75 — баланс между живостью и точностью следования стилю.
Умеет писать с нуля, переписывать чужой текст, генерировать A/B варианты.

**Входные данные:**
- `style_profile.json`
- Тема / черновик от пользователя
- Фактура от Ресерчера
- Замечания от Критика (при доработке)

---

### R-5. Критик (Critic)

| Параметр    | Значение                              |
|-------------|---------------------------------------|
| Модель      | `claude-haiku-4-5-20251001`           |
| Температура | `0.0`                                 |
| Роль        | Оценка качества, решение о ревизии    |

**Описание:**
Оценивает пост по шкале 1-10, выносит вердикт APPROVE/REVISE.
Haiku — потому что только оценивает, не пишет. Быстро и дёшево.
Температура 0 — оценка должна быть детерминированной.

**Критерии (по 2 балла каждый):**
- Соответствие стилю (лексика, тон, структура)
- Первое предложение держит внимание
- Нет воды и общих слов
- Длина подходит для Telegram
- Нет AI-паттернов

**Выход:** `{ score: 1-10, verdict: APPROVE|REVISE, issues: [], suggestions: [] }`
APPROVE если score ≥ 7, REVISE если < 7. Максимум 2 итерации.

---

### R-6. Редактор (Editor)

| Параметр    | Значение                              |
|-------------|---------------------------------------|
| Модель      | `claude-sonnet-4-6`                   |
| Температура | `0.4`                                 |
| Роль        | Правки: короче / хлёстче / человечнее |

**Описание:**
Применяет конкретный тип правки к тексту. Работает по замечаниям
Критика или по прямой инструкции пользователя (кнопки или текст).
Отделён от Критика — не может оценивать то, что сам же написал.

**Режимы работы:**
- `shorter` — сократить на 30-40% без потери смысла
- `punchier` — усилить первое предложение, убрать мягкие формулировки
- `human` — убрать AI-паттерны, добавить разговорность
- `custom` — свободная инструкция пользователя

---

### R-7. Контент-планер (Content Planner)

| Параметр    | Значение                              |
|-------------|---------------------------------------|
| Модель      | `claude-sonnet-4-6`                   |
| Температура | `0.6`                                 |
| Роль        | Создание контент-планов               |

**Описание:**
Создаёт контент-план на N дней на основе тематики канала из
`style_profile.json`. Учитывает баланс форматов (мнение/кейс/лайфхак/
провокация) и недавно опубликованные темы.

**Выход:** список `{ date, topic, format, angle }` — тема + угол подачи на каждый день.

## Часть II — Dev-агенты (строят бот, не работают в продакшне)

### Матрица Dev-агентов

| Агент | Модель | Темп. | Роль |
|---|---|---|---|
| **Оркестратор** | `claude-opus-4-6` | 0.3 | Координация, точка входа при разработке |
| **Архитектор** | `claude-opus-4-6` | 0.2 | Структура проекта, API-контракты |
| **Разработчик** | `claude-sonnet-4-6` | 0.1 | Весь production-код, хэндлеры, БД |
| **Декомпозитор** | `claude-haiku-4-5-20251001` | 0.2 | Разбивка задач на атомарные шаги |
| **Security Agent** | `claude-sonnet-4-6` | 0.0 | Аудит: секреты, инъекции, доступ |
| **Дебагер** | `claude-sonnet-4-6` | 0.1 | Локализация и фикс ошибок |
| **Тестировщик** | `claude-sonnet-4-6` | 0.1 | Unit/integration тесты, smoke после деплоя |

### Dev Flow: Новая фича
```
Запрос → Оркестратор
  → Декомпозитор (разбивает на задачи)
  → Архитектор (проектирует структуру)
  → Разработчик (пишет код)
  → Security Agent (аудит)
  → Тестировщик (проверяет)
  → Оркестратор → Готово
```

### Dev Flow: Баг-репорт
```
Баг → Оркестратор
  → Дебагер (локализует причину, фиксит)
  → Тестировщик (проверяет фикс)
  → Оркестратор → Готово
```

## Стек проекта

| Слой              | Технология                                                   |
|-------------------|--------------------------------------------------------------|
| Telegram Bot      | `aiogram 3.x` (async, FSM встроен)                          |
| LLM API           | `OpenRouter` (openai-совместимый, доступ к Claude)           |
| LLM — роутинг     | `claude-haiku-4-5-20251001` (Диспетчер, Критик — дёшево)    |
| LLM — контент     | `claude-sonnet-4-6` (Копирайтер, Редактор, Аналитик)        |
| Хранилище         | `SQLite` + `aiosqlite` (посты, контент-план, примеры)        |
| Style Profile     | `data/style_profile.json` (ДНК стиля, обновляется редко)    |
| Конфиг            | `.env` + `pydantic-settings`                                 |
| Среда             | Python 3.11+                                                 |
| Деплой            | Docker + docker-compose / Railway                            |
