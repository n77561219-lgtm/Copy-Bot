## 1. Что такое Copy-Bot

Telegram-бот для контент-менеджеров: генерирует посты в стиле автора, создаёт контент-планы, ищет тренды, публикует в канал. Архитектура: aiogram 3.x + PostgreSQL + OpenRouter (LLM) + Apify (YouTube) + pytrends (Google Trends).

---

## 2. Быстрый старт

### 2.1 Требования

- Docker и Docker Compose
- Аккаунт на [openrouter.ai](https://openrouter.ai) с балансом (минимум $1)
- Telegram-бот (токен от @BotFather)
- PostgreSQL (или Docker-контейнер)

### 2.2 Установка локально

```bash
git clone https://github.com/n77561219-lgtm/Copy-Bot.git
cd Copy-Bot
cp .env.example .env   # если нет — создай вручную
```

Заполни `.env`:

```env
TELEGRAM_BOT_TOKEN=<токен от @BotFather>
TELEGRAM_ALLOWED_USERS=<твой Telegram user_id, узнать у @userinfobot>
OPENROUTER_API_KEY=<ключ с openrouter.ai>
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
MODEL_HAIKU=anthropic/claude-3-5-haiku
MODEL_SONNET=anthropic/claude-3-5-sonnet
MODEL_IMAGE=google/gemini-2.5-flash-image
MODEL_TRENDS=perplexity/sonar
DATABASE_URL=postgresql://copybot_user:copybot_pass_2026@localhost:5432/copybot
UPLOADS_DIR=data/uploads
APIFY_TOKEN=<опционально, нужен для поиска по теме>
```

### 2.3 Запуск через Docker

```bash
docker compose up -d --build
docker logs copy-bot --tail 30
```

Ожидаемый вывод:
```
[INFO] __main__: Bot started
[INFO] aiogram.dispatcher: Start polling
[INFO] aiogram.dispatcher: Run polling for bot @YourBot
```

### 2.4 Запуск локально (без Docker)

```bash
pip install -r requirements.txt
python -m bot.main
```

### 2.5 База данных

При старте бот автоматически создаёт все таблицы через `init_db()`. Миграции не нужны.

| Таблица | Назначение |
|---|---|
| `style_examples` | Примеры постов для обучения стилю |
| `posts` | Сохранённые сгенерированные посты |
| `content_plan` | Контент-план |
| `user_preferences` | Настройки пользователя (key-value) |
| `subscriptions` | Подписки (free / trial / basic / standard / pro) |
| `usage_log` | Лог действий (post_generated, image_generated, payment) |
| `referrals` | Реферальные связи |
| `scheduled_posts` | Очередь отложенных публикаций |
| `schedule_slots` | Временны́е слоты публикаций пользователя (UTC) |
| `renewal_notifications` | Дедупликация напоминаний о продлении |
| `payments` | Лог платежей с суммой в ₽ |
| `token_log` | Лог токенов по агентам для контроля расходов |

## 5. Тест-план

### TC-01: Запуск и онбординг (новый пользователь)

**Шаги:**
1. Отправь `/start`
2. Бот показывает приветствие с указанием пробного периода и онбординг-кнопки:
   - `📤 Загрузить стиль сейчас` (callback: `onboarding:upload`)
   - `➡️ Пропустить` (callback: `onboarding:skip`)
3. Повторно `/start` → только «С возвращением!»

**Ожидаемый результат:**
- `subscriptions`: `plan='trial'`, `status='active'`, `expires_at = now + 5 days`
- Онбординг-кнопки работают

### TC-16: Автопродление подписки

**Предусловие:** Пользователь с `auto_renew='1'`, подписка истекает через 1 день.

**Ожидаемый результат (через renewal_notification_loop):**
- Бот присылает текст-напоминание + инвойс напрямую (без ссылки на планы)
- После оплаты → `activate_subscription` продлевает период
- `renewal_notifications`: запись `(user_id, days_left=1)` предотвращает дублирование

**Если `auto_renew='0'`:**
- Бот присылает только текст со ссылкой на тарифы (без инвойса)

**Моменты проверки:** За 3 дня, 1 день и в день истечения (проверяется каждый час).

```sql
-- Принудительный тест: установи expires_at на завтра
UPDATE subscriptions SET expires_at = NOW() + INTERVAL '1 day' WHERE user_id = <id>;
-- Очисти notification чтобы снова сработало
DELETE FROM renewal_notifications WHERE user_id = <id>;
```

### TC-17: Feature gates и monthly limits

**Шаги:**

**Feature gates:**
1. На Free-тарифе нажми «🔥 Тренды» → сообщение «Функция недоступна на вашем тарифе»

   ```sql
   -- Для тарифа basic (лимит 30 постов/мес)
   INSERT INTO usage_log (user_id, action, created_at)
   SELECT <id>, 'post_generated', NOW() - (generate_series(1,30) * INTERVAL '1 hour')
   ```
2. Попробуй сгенерировать пост → сообщение о превышении лимита

**Ожидаемый результат:**
- `subscription_middleware.py` блокирует действие и показывает paywall с `plans_kb()`
- После апгрейда — функция становится доступна

### TC-19: Подключение канала (3-шаговый онбординг)

**Предусловие:** Бот добавлен в канал как администратор с правом публикации. Тариф Standard+.

**Шаги:**
1. `/channel` или «⚙️ Настройки» → «🔗 Подключить канал»

   - Бот отправит тестовое сообщение и удалит его
3. **Шаг 2/3** — выбери часовой пояс из кнопок или введи вручную (например `3` для UTC+3)
4. **Шаг 3/3** — введи слоты через запятую: `10:00, 18:00` или `/skip`

**Ожидаемый результат:**
- `user_preferences`: `publish_channel`, `timezone_offset`
- `schedule_slots`: записи в UTC (с учётом смещения)

**Ожидаемые ошибки:**
- Бот не добавлен → «❌ Бот не может отправить сообщение»
- Неверный ID → «❌ Неверный chat ID»
- Нет Standard+ → сообщение о необходимости апгрейда

### TC-20: Расписание публикаций

**Предусловие:** Канал подключён (TC-19), слоты настроены.

**Шаги — настройка:**
1. «⏰ Расписание» → показывает слоты, статистику, кнопки управления
2. «➕ Добавить слот» → введи `10:00`
3. «🗑 Удалить слот» → выбери слот

**Шаги — добавление поста:**
1. Сгенерируй пост (канал настроен → кнопки 📢 и ⏰ видны)
2. Нажми «⏰ В очередь»
   - Если слоты есть: предлагает ближайший слот с кнопками «✅ Да» / «🕐 Другое время»
   - Если нет слотов: просит ввести дату вручную
3. Вручную: `завтра в 10:00` / `08.04 15:30` / `09.04.2026 09:00`

**Шаги — управление:**
1. «⏰ Расписание» → список постов с кнопками 🗑
2. «⏸ Пауза» → публикации останавливаются
3. «▶️ Возобновить» → публикации возобновляются

**Быстрый тест планировщика:**
```sql
INSERT INTO scheduled_posts (user_id, content, channel_id, scheduled_at)
VALUES (<id>, 'Тест расписания', '@channel', NOW() - INTERVAL '1 minute');
```
Подожди до 60 секунд — пост появится в канале.

**Ожидаемый результат:**
- `scheduled_posts`: `status='published'` после публикации
- При 3 неудачных попытках: `status='failed'`, уведомление пользователю

**Лимит слотов по тарифу:** Free=0, Basic=1, Standard=3, Pro=∞

## 6. Типичные ошибки и решения

| Ошибка | Причина | Решение |
|---|---|---|
| `ConnectionRefusedError: [Errno 111]` | Нет подключения к PostgreSQL | Проверь `DATABASE_URL`. Для Docker: используй IP контейнера, не `localhost` |
| `InsufficientPrivilegeError: permission denied for schema public` | Пользователь БД без прав | `GRANT ALL ON SCHEMA public TO copybot_user;` |
| `TelegramConflictError` | Два экземпляра бота | `taskkill -F -IM python.exe` (Windows) / `pkill -f bot.main` (Linux) |
| `402` при генерации | Нет баланса OpenRouter | Пополни на openrouter.ai |
| `Cannot extract image` | Неверная модель Gemini | `MODEL_IMAGE=google/gemini-2.5-flash-image` |
| `No Apify token` | Пустой APIFY_TOKEN | Добавь токен или не используй «Поиск по теме» |
| Кнопки 📢/⏰ не видны под постом | Канал не настроен | Пройди TC-19 |
| Тренды недоступны на Free | Feature gate | Нужен Basic+ |
| Автопубликация не отправляет | Очередь на паузе | «▶️ Возобновить» в расписании |
| token_log пустой | user_id не передаётся в агент | Генерация должна идти через `_generate_post()` с явным `user_id` |

## 7. Проверка БД вручную

```bash
# Зайти в PostgreSQL (Docker)
docker exec -it root-postgres-1 psql -U postgres -d copybot

# Пользователи и планы
SELECT user_id, plan, status, expires_at FROM subscriptions ORDER BY expires_at DESC;

# Выручка
SELECT plan, period, amount_rub, created_at FROM payments ORDER BY created_at DESC LIMIT 10;

# Токены по агентам
SELECT agent, model, SUM(input_tokens) AS inp, SUM(output_tokens) AS out, COUNT(*) AS calls
FROM token_log GROUP BY agent, model ORDER BY out DESC;

# Рефералы
SELECT referrer_id, invited_id, bonus_given, created_at FROM referrals ORDER BY created_at DESC;

# Очередь публикаций
SELECT id, topic, scheduled_at, status, attempts FROM scheduled_posts WHERE user_id = <id>;

# Настройки пользователя
SELECT key, value FROM user_preferences WHERE user_id = <id> ORDER BY key;

\q
```
