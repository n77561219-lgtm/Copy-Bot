# TESTING.md — Руководство тестировщика Copy-Bot

> Документ составлен на основе исходного кода. Все данные — реальные: команды, состояния FSM, callback-строки, схемы БД.

---

## 1. Что такое Copy-Bot

Telegram-бот для контент-менеджеров: генерирует посты в стиле автора, создаёт контент-планы, ищет тренды, публикует в канал. Архитектура: aiogram 3.x + PostgreSQL + OpenRouter (LLM) + Apify (YouTube) + pytrends (Google Trends).

---

## 2. Быстрый старт

### 2.1 Требования

- Docker и Docker Compose
- Аккаунт на [openrouter.ai](https://openrouter.ai) с балансом (минимум $1)
- Telegram-бот (токен от @BotFather)
- PostgreSQL (или Docker-контейнер)

### 2.2 Установка локально

```bash
git clone https://github.com/n77561219-lgtm/Copy-Bot.git
cd Copy-Bot
cp .env.example .env   # если нет — создай вручную
```

Заполни `.env`:

```env
TELEGRAM_BOT_TOKEN=<токен от @BotFather>
TELEGRAM_ALLOWED_USERS=<твой Telegram user_id, узнать у @userinfobot>
OPENROUTER_API_KEY=<ключ с openrouter.ai>
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
MODEL_HAIKU=anthropic/claude-3-5-haiku
MODEL_SONNET=anthropic/claude-3-5-sonnet
MODEL_IMAGE=google/gemini-2.5-flash-image
MODEL_TRENDS=perplexity/sonar
DATABASE_URL=postgresql://copybot_user:copybot_pass_2026@localhost:5432/copybot
UPLOADS_DIR=data/uploads
APIFY_TOKEN=<опционально, нужен для поиска по теме>
```

### 2.3 Запуск через Docker

```bash
docker compose up -d --build
docker logs copy-bot --tail 30
```

Ожидаемый вывод:
```
[INFO] __main__: Bot started
[INFO] aiogram.dispatcher: Start polling
[INFO] aiogram.dispatcher: Run polling for bot @YourBot
```

### 2.4 Запуск локально (без Docker)

```bash
pip install -r requirements.txt
python -m bot.main
```

### 2.5 База данных

При старте бот автоматически создаёт все таблицы через `init_db()`. Таблицы:

| Таблица | Назначение |
|---|---|
| `style_examples` | Примеры постов для обучения стилю |
| `posts` | Сохранённые сгенерированные посты |
| `content_plan` | Контент-план |
| `user_preferences` | Настройки пользователя (key-value) |
| `subscriptions` | Подписки (trial / paid) |
| `usage_log` | Лог действий |
| `referrals` | Реферальные связи |
| `scheduled_posts` | Очередь отложенных публикаций |
| `schedule_slots` | Дефолтные временны́е слоты пользователя (UTC) |

## 4. Тест-план

### TC-01: Запуск и онбординг

**Шаги:**
1. Отправь `/start`
2. Если пользователь новый (нет загруженного стиля) — бот должен показать два сообщения:
   - Приветствие с описанием функций и кол-вом дней пробного периода
   - Онбординг-сообщение с кнопками `📤 Загрузить стиль сейчас` и `➡️ Пропустить`
3. Повторно отправь `/start` — бот должен показать только «С возвращением!»

**Ожидаемый результат:**
- В базе появилась запись в `subscriptions` с `plan='trial'`, `status='active'`, `expires_at = now + 7 days`
- Кнопки работают: `onboarding:upload` показывает инструкцию, `onboarding:skip` закрывает онбординг

---

### TC-02: Загрузка стиля (JSON)

**Подготовка:** Экспортируй любой Telegram-канал через Telegram Desktop → «Экспорт истории» → JSON.

**Шаги:**
1. Отправь файл `result.json` боту
2. Бот должен показать: «📥 Загружаю файл...» → «🔍 Парсю посты...» → «📊 Найдено N постов. Анализирую стиль...»
3. Финальное сообщение: тональность и темы

**Ожидаемый результат:**
- Файл `data/style_profiles/{user_id}_main.json` создан
- В `style_examples` появились записи
- В `user_preferences` ключ `active_profile = 'main'`

**Лимиты:** Файл до 10 МБ, форматы: `.json`, `.md`, `.txt`

---

### TC-03: Загрузка стиля (Tone of Voice .md)

**Подготовка:** Создай файл `style.md` с описанием стиля: тон, любимые слова, запрещённые фразы.

**Шаги:**
1. Отправь `.md` файл боту
2. Бот должен написать: «📖 Вижу документ Tone of Voice. Читаю напрямую...»

**Ожидаемый результат:** Профиль создан без парсинга постов, напрямую из документа.

---

### TC-04: Несколько профилей стиля

**Шаги:**
1. Загрузи первый файл → профиль `main` создан автоматически
2. Загрузи второй файл → бот предложит кнопки: «📁 Обновить «main»» и «➕ Новый профиль»
3. Нажми «➕ Новый профиль» → введи имя, например `работа`
4. Нажми «🧠 Мой стиль» → «🔄 Сменить профиль стиля» → выбери профиль

**Ограничения:** Максимум 3 профиля. Имя: до 20 символов, только буквы/цифры/-_.

**Ожидаемый результат:**
- Файлы `{user_id}_main.json` и `{user_id}_работа.json` в `data/style_profiles/`
- В `user_preferences` ключ `active_profile` меняется при переключении

### TC-05: Генерация поста

**Предусловие:** Стиль загружен (TC-02 или TC-03).

**Шаги:**
1. Нажми «✏️ Написать пост»
2. Введи тему, например: «Как я научился работать без выгорания»
3. Дождись прохода: «🔍 Исследую тему...» → «✍️ Пишу пост...» → «🔎 Проверяю качество...»

**Ожидаемый результат:**
- Пост в стиле автора с оценкой (если `show_score = yes`)
- Формат оценки: `🟢 Оценка: 8.5/10  (стиль: 9, контент: 8, вовлечение: 8)`
- Кнопки редактирования под постом

**Кнопки под постом:**

| Кнопка | callback_data | Что делает |
|---|---|---|
| ✂️ Короче | `edit:shorter` | Сокращает на 30-40% |
| 📝 Длиннее | `edit:longer` | Расширяет на 30-40% |
| 🫀 Человечнее | `edit:human` | Убирает AI-паттерны |
| 🔥 Хлёстче | `edit:punchier` | Делает более цепляющим |
| ✏️ Грамматика | `edit:grammar` | Только грамматика |
| 🔄 Перегенерировать | `post:regenerate` | Новый вариант на ту же тему |
| 🖼 Картинка | `image:generate` | Генерирует изображение 16:9 |
| ✅ Готово | `post:save` | Сохраняет в `posts` таблицу |

### TC-08: Настройки

**Шаги:**
1. Нажми «⚙️ Настройки»
2. Нажми «📏 Длина поста» — циклически переключается: `short` → `medium` → `long`
3. Нажми «📊 Оценка качества» — переключается: `yes` ↔ `no`
4. Нажми «🔄 Итерации правки» — переключается: `1` ↔ `2`
5. Нажми «🔗 Подключить канал» — запускает 3-шаговый флоу настройки канала (см. TC-16)

**Ожидаемый результат:** Значения сохраняются в `user_preferences`:

| Ключ | Допустимые значения | По умолчанию |
|---|---|---|
| `post_length` | `short`, `medium`, `long` | `medium` |
| `show_score` | `yes`, `no` | `yes` |
| `critic_iters` | `1`, `2` | `2` |
| `publish_channel` | `@username` или `-100xxxxxxxxx` | — |
| `timezone_offset` | целое число от -12 до 14 | `0` (UTC) |
| `queue_paused` | `0`, `1` | `0` |

### TC-16: Подключение канала (3-шаговый онбординг)

**Предусловие:** Бот добавлен в Telegram-канал как администратор с правом публикации.

**Шаги:**
1. Отправь `/channel` или нажми «⚙️ Настройки» → «🔗 Подключить канал»
2. **Шаг 1/3** — введи `@username_канала` или `-100xxxxxxxxx`
   - Бот отправит тестовое сообщение и сразу удалит его
   - Успех: «✅ Канал подключён! Шаг 2/3...»
3. **Шаг 2/3** — выбери часовой пояс из кнопок или нажми «✍️ Ввести вручную»
   - Вручную: введи число, например `3` для UTC+3
   - Или нажми «⏭ Пропустить» — сохранится UTC+0
4. **Шаг 3/3** — введи времена публикации через запятую: `10:00, 18:00`
   - Или `/skip` чтобы пропустить
5. Бот показывает сводку: канал, часовой пояс, слоты

**Ожидаемый результат:**
- `user_preferences`: `publish_channel`, `timezone_offset`
- `schedule_slots`: записи с временами в UTC (с учётом смещения)
- Сводка: `📢 @mychannel · UTC+3 · слоты: 07:00, 15:00 UTC`

**Ожидаемые ошибки:**
- Бот не добавлен → `TelegramForbiddenError` → «❌ Бот не может отправить сообщение»
- Неверный ID → `TelegramBadRequest` → «❌ Неверный chat ID»
- Неверный timezone вручную → «❌ Введи число от -12 до 14»

### TC-17: Расписание публикаций

**Предусловие:** Канал подключён (TC-16), слоты настроены.

**Шаги — настройка очереди:**
1. Нажми «⏰ Расписание» в главном меню
2. Бот показывает: слоты UTC, статистику очереди, статус (активна/пауза)
3. Нажми «➕ Добавить слот времени» → введи `10:00` → слот сохранится в UTC
4. Нажми «🗑 Удалить слот» → введи время → слот удалится

**Шаги — добавление поста в очередь:**
1. Сгенерируй пост при настроенном канале
2. Под постом появятся две кнопки: «📢 Опубликовать сейчас» и «⏰ В очередь»
3. Нажми «⏰ В очередь»
   - Если слоты настроены: бот предлагает ближайший свободный слот с кнопками «✅ Да» / «🕐 Другое время»
   - Если слотов нет: бот просит ввести дату вручную
4. Вручную: введи `завтра в 10:00` или `08.04 15:30` или `09.04.2026 09:00`
5. Бот подтверждает: «⏰ Пост добавлен в очередь на 08 Apr 10:00 UTC»

**Шаги — управление очередью:**
1. «⏰ Расписание» → список постов с датами и кнопками 🗑
2. Нажми 🗑 напротив поста → пост удалён из очереди
3. «⏸ Пауза очереди» → публикации останавливаются
4. «▶️ Возобновить» → публикации возобновляются

**Шаги — автопубликация планировщиком:**
1. Добавь пост в очередь на время в прошлом (через прямой INSERT в БД) для немедленного теста:
   ```sql
   INSERT INTO scheduled_posts (user_id, content, channel_id, scheduled_at)
   VALUES (<id>, 'Тест расписания', '@channel', NOW() - INTERVAL '1 minute');
   ```
2. Подожди до 60 секунд
3. Пост появится в канале

**Ожидаемый результат в БД:**
- `scheduled_posts`: `status='published'` после публикации
- При ошибке (3 попытки): `status='failed'`, пользователь получает уведомление

**Проверка в БД:**
```sql
SELECT id, topic, scheduled_at, status, attempts FROM scheduled_posts WHERE user_id = <id>;
SELECT time_utc FROM schedule_slots WHERE user_id = <id>;
```

## 5. Типичные ошибки и решения

| Ошибка | Причина | Решение |
|---|---|---|
| `ConnectionRefusedError: [Errno 111]` | Бот не может подключиться к PostgreSQL | Проверь `DATABASE_URL` в `.env`. Для Docker: используй IP контейнера, не localhost |
| `InsufficientPrivilegeError: permission denied for schema public` | Пользователь БД без прав | `GRANT ALL ON SCHEMA public TO copybot_user;` в psql |
| `TelegramConflictError` | Два экземпляра бота запущены | Убей все процессы Python: `taskkill -F -IM python.exe` (Windows) или `pkill -f bot.main` (Linux) |
| `402` в логах при генерации | Нет баланса OpenRouter | Пополни баланс на openrouter.ai |
| `Cannot extract image` | Неверный формат ответа Gemini | Убедись что `MODEL_IMAGE=google/gemini-2.5-flash-image` |
| `No Apify token` | Пустой APIFY_TOKEN | Добавь токен в `.env` или не используй «Поиск по теме» |
| Бот не реагирует | Истёкшая подписка + paywall | Продли подписку или установи `expires_at` в будущее |
| Стиль не загружается | Файл > 10 МБ или неверный формат | Только `.json`, `.md`, `.txt`, до 10 МБ |
