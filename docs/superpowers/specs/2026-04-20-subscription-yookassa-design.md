# Дизайн: система платных подписок через ЮКасса

**Дата:** 2026-04-20  
**Проект:** Copy-Bot  
**Статус:** approved

---

## Контекст

Telegram-бот для написания постов. Текущая оплата — Telegram Stars — заменяется на ЮКасса (рубли, карта/СБП). Добавляется настоящее автосписание: при первой оплате карта сохраняется, при продлении бот списывает деньги без участия пользователя.

---

## Решения

| Вопрос | Ответ |
|---|---|
| Провайдер оплаты | ЮКасса (заменяет Telegram Stars полностью) |
| Автосписание | Настоящее — ЮКасса списывает по сохранённой карте |
| Webhook-сервер | aiohttp встроен в бот-процесс (один Docker-контейнер) |
| Схема БД | `subscriptions` + новая `payment_methods` + переработанная `payments` |
| Лимиты при истечении | Сгорают немедленно, grace-период отсутствует |
| Апгрейд плана | Новый период начинается с даты оплаты, дни старого плана сгорают |
| Неиспользованные ресурсы | Сгорают при истечении или смене плана |

## Схема БД

### Изменения в `subscriptions`

Добавить колонки:

```
payment_method_id    BIGINT → payment_methods(id)   -- привязанная карта
next_renewal_at      TIMESTAMPTZ                     -- когда инициировать автосписание
renewal_status       TEXT DEFAULT 'ok'               -- ok | failed | cancelled
billing_period_start TIMESTAMPTZ                     -- сбрасывается при каждом успешном платеже
```

Допустимые значения `status`: `active` | `expired` | `cancelled`

### Новая таблица `payment_methods`

| Колонка | Тип | Описание |
|---|---|---|
| id | BIGSERIAL PK | |
| user_id | BIGINT | |
| yookassa_method_id | TEXT UNIQUE | ID метода из ЮКасса |
| type | TEXT | bank_card \| sbp \| yoo_money |
| brand | TEXT | Visa \| MasterCard \| Mir |
| last4 | TEXT | последние 4 цифры карты |
| is_default | BOOLEAN | основная карта пользователя |
| is_active | BOOLEAN | false = карта протухла или отклонена |
| created_at | TIMESTAMPTZ | |

### Переработанная таблица `payments`

| Колонка | Тип | Описание |
|---|---|---|
| id | BIGSERIAL PK | |
| user_id | BIGINT | |
| yookassa_payment_id | TEXT UNIQUE | главный idempotency-ключ |
| plan | TEXT | |
| period | TEXT | month \| year |
| amount_rub | NUMERIC(10,2) | сумма берётся из PLANS на сервере |
| status | TEXT | pending \| succeeded \| failed \| refunded \| cancelled |
| payment_method_id | BIGINT → payment_methods | |
| is_renewal | BOOLEAN | false = первый платёж, true = автопродление |
| idempotence_key | TEXT UNIQUE | UUID для ЮКасса API |
| created_at | TIMESTAMPTZ | |
| updated_at | TIMESTAMPTZ | |

### Как требования закрываются схемой

| Требование | Решение |
|---|---|
| Webhook не активирует дважды | `yookassa_payment_id UNIQUE` — повторный INSERT упадёт, статус проверяется до обработки |
| Цена на сервере | `amount_rub` берётся из `PLANS[plan][price_rub]`, не из запроса клиента |
| Сброс счётчика при продлении | `UPDATE subscriptions SET billing_period_start = NOW()` |
| Раздельный учёт free/paid | Платные: счёт с `billing_period_start`; Free: с начала календарного месяца |
| Фантомный остаток | После истечения лимиты пересчитываются по новому плану немедленно |

## Поток оплаты

### Первый платёж (новая карта)

1. Пользователь нажимает «Оплатить»
2. Бот создаёт payment в ЮКасса с `save_payment_method=true`, получает ссылку
3. Бот отправляет ссылку пользователю в чат
4. Пользователь платит на странице ЮКасса — карта сохраняется
5. ЮКасса присылает `payment.succeeded` на webhook
6. Бот активирует подписку, сохраняет карту, обнуляет `billing_period_start`

### Автопродление (сохранённая карта)

1. За 3 дня до `expires_at` фоновая задача создаёт payment в ЮКасса с `payment_method_id`
2. Пользователь не участвует
3. `payment.succeeded` → подписка продлевается, пользователь получает уведомление
4. `payment.canceled` → немедленный перевод на Free, уведомление об ошибке

## Webhook-события

| Событие ЮКасса | Действие бота |
|---|---|
| `payment.succeeded` | Сохранить/обновить `payment_methods`, обновить `payments` → succeeded, активировать подписку, сбросить `billing_period_start`, уведомить пользователя |
| `payment.canceled` | Обновить `payments` → failed/cancelled; если это автосписание (`is_renewal=true`) — немедленно Free + уведомление; если первый платёж — только уведомление об ошибке, подписка не меняется; пометить `payment_method.is_active=false` если карта отклонена |
| `refund.succeeded` | Обновить `payments` → refunded; при полном возврате — Free + очистить `payment_method_id`; уведомить |

События `payment.waiting_for_capture` и `payment.pending` не обрабатываются (используется автозахват).

### Безопасность webhook

1. Секретный путь: `/yookassa/webhook/{WEBHOOK_SECRET}` — случайный UUID в `.env`
2. IP-фильтрация по опубликованным диапазонам ЮКасса
3. Валидация тела через Python SDK (`yookassa.WebhookNotification`)
4. Идемпотентность: перед обработкой — проверка `status` по `yookassa_payment_id`; если уже `succeeded` → 200 OK без действий

## Логика открытия и закрытия доступа

### Открытие (при `payment.succeeded`)

1. Установить план и `expires_at`
2. Сохранить карту в `payment_methods`
3. Обнулить `billing_period_start`
4. Уведомить пользователя

### Закрытие — две отдельные задачи

**Задача 1: инициация автосписания (раз в час)**
Находит платные подписки, у которых `next_renewal_at <= NOW()` и `renewal_status = 'ok'`.
Создаёт платёж в ЮКасса с сохранённой картой. Результат приходит через webhook:
- `payment.succeeded` → подписка продлевается, `next_renewal_at` сдвигается вперёд
- `payment.canceled` → немедленно Free + уведомление об ошибке карты

**Задача 2: принудительное истечение (каждые 5 минут)**
Находит платные подписки, у которых `expires_at < NOW()` и `status = 'active'`.
К этому моменту автосписание уже либо прошло (и подписка продлена), либо не прошло (и пользователь уже на Free после webhook). Задача — страховочная сетка для случаев когда карты нет или автосписание не запускалось.
Действие: перевод на Free + одно сообщение «подписка закончилась» с кнопкой «Оформить снова».

Истечение пробного периода (trial) обрабатывается отдельно — существующим `ensure_free_plan` в middleware, без уведомления.

Grace-период отсутствует. Ограничения применяются немедленно.

### Счётчики использования

- **Платная подписка:** считается с `billing_period_start`, сбрасывается при каждой успешной оплате
- **Free:** считается с начала календарного месяца, сбрасывается первого числа автоматически
- При истечении подписки остаток сгорает, новый счётчик идёт по правилам Free

## Краевые сценарии

### Проблемы с картой

| Сценарий | Действие |
|---|---|
| Карта заблокирована / недостаточно средств | Free + уведомление + кнопка «Обновить карту» |
| Карта истекла | То же + `payment_method.is_active=false`; при следующей оплате вводится новая |
| Пользователь обновил карту и оплатил вручную | Подписка восстанавливается, новая карта становится основной |

### Возвраты

- **Полный возврат:** подписка закрывается, карта отвязывается, пользователь → Free
- **Частичный возврат:** только фиксируется в `payments`, подписка не меняется

### Двойной платёж

Повторное уведомление от ЮКасса игнорируется по `yookassa_payment_id`. Фактический дубль (пользователь заплатил дважды) — возврат вручную через личный кабинет ЮКасса.

### Апгрейд плана

Новый период начинается с даты оплаты Pro. Пользователь платит полную стоимость за полный новый период. Дни оставшегося предыдущего плана сгорают. На экране оплаты явно указано: «подписка начнётся с сегодня».

### Технические сбои

| Сбой | Решение |
|---|---|
| Бот недоступен при webhook | ЮКасса повторяет уведомление — обработается при следующей попытке |
| Бот активировал подписку, но не отправил сообщение | Подписка активна, пользователь видит статус в /profile |
| Бот упал во время обработки webhook | БД-транзакция откатится — следующий webhook обработается корректно |
